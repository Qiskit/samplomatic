Made `BoxKey` hashing deterministic across Python processes by using SHA-256 instead of the built-in `hash()`, which is randomized by `PYTHONHASHSEED`. This caused `AddInjectNoise` to generate different `ref` names for identical boxes across runs, since it derived short hash keys from `BoxKey.__hash__`. Also fixed a bug in `AddInjectNoise._get_ref()` where the computed ref was not being cached correctly.
